# Konvey — интерактивное демо

> AI-конструктор для SMB: настройка CRM, шаблоны сайтов/mini-apps/ботов, маркетплейс интеграций — в одном окне.

**Живое демо:** https://donyado.github.io/konvey/

Single-page интерактивный прототип платформы для показа партнёрам и первым клиентам. Без сборки, без сервера — открывается двойным кликом или как статический сайт на GitHub Pages.

## Что можно потыкать

- **Лендинг** — маркетинговая страница с фичами и тарифами
- **AI-настройщик CRM** — чат с scripted-моделью, которая понимает команды на русском:
  - «Добавь статус "Согласование" в воронку»
  - «Создай поле "Размер обуви" в контактах»
  - «Настрой воронку под фитнес-клуб» / барбершоп / фотостудию / стоматологию / курсы
  - «Покажи статистику», «Добавь тег VIP»
  Каждая мутация показывается как **diff-карточка** с кнопкой Apply — точь-в-точь как в боевой версии.
- **Интеграции** — подключение amoCRM / Битрикс24 / Yclients / Telegram-бота через OAuth-модалку
- **Шаблоны** — каталог с фильтрами
- **Конструктор шаблона** — live-preview лендинга справа, AI-подсказки и ручные настройки слева
- **Маркетплейс** — 10 готовых рецептов + 3-шаговый визард установки
- **Биллинг** — тарифы, usage-бары, история платежей
- **Журнал** — аудит-лог с фильтрами
- **Настройки** — workspace, безопасность, команда

Всё состояние сохраняется в `localStorage` — примененные мутации, установленные рецепты, подключённые интеграции остаются между визитами. Сбросить: **Настройки → «Сбросить демо-данные»**.

## Как это работает

- Один HTML-файл (`index.html`, ~110 KB) без сборки и зависимостей.
- Ванильный JS: template literals + event delegation через `data-action`.
- Tailwind CSS через Play CDN.
- **Scripted AI** — детерминированный парсер команд на клиенте. Никаких API-ключей, работает оффлайн после первой загрузки.

## Что "не по-настоящему"

Это демо для показа, не боевая версия. Не работает без backend:
- Реальный Claude API (в боевой версии — через серверный релей)
- Настоящий OAuth в amoCRM/Битрикс24/Yclients
- Реальные Telegram-боты
- Реальные платежи Stripe/ЮKassa

Всё остальное — интерфейс, логика confirmation gate, live-preview конструктора, симулятор маркетплейса — работает так же, как будет работать в проде.

## Деплой на GitHub Pages

1. **Settings → Pages** в репозитории.
2. **Source:** Deploy from a branch
3. **Branch:** `main`, **folder:** `/ (root)`
4. Save. Через 30–60 секунд демо появится по адресу `https://<username>.github.io/<repo>/`.

## Локальный запуск

Просто открыть `index.html` двойным кликом — работает.

Или через локальный HTTP-сервер (лучше — так проще отлаживать):
```
npx -y serve .
# → http://localhost:3000
```

## Полный продукт

Демо — витрина. Основной проект: Next.js 15 монорепо с настоящими коннекторами CRM, Claude tool use loop, Supabase, шаблонизатором и bot-runtime. Пишите в личку — покажу.
